## Context

Clincus is a Go CLI + Svelte 5 web dashboard for managing AI coding sessions in Incus containers. It provides container isolation, session persistence, workspace mounting, snapshots, and resource limits.

## Patterns

### Error Handling

- Go: return `error` values; use `fmt.Errorf("context: %w", err)` for wrapping
- Never panic in library code; panics are only acceptable in `main()` for truly unrecoverable situations
- CLI commands should print user-friendly errors via Cobra's `RunE` pattern

### Testing

- **Go unit tests:** `make test` — 56 tests across 7 packages, standard `testing` package with table-driven tests
- **Go integration tests:** require running Incus with `clincus` image built
- **Python integration tests:** `pytest tests/` — require Incus + built image
- **Python linting:** `ruff check tests/ && ruff format --check tests/`
- **Go linting:** `make lint` — golangci-lint v2 (`.golangci.yml`)
- Test files live alongside source: `foo_test.go` next to `foo.go`

### Code Style

- Go: standard `gofmt`/`goimports`, golangci-lint v2
- Frontend: Svelte 5 with TypeScript, Vite bundler
- Python tests: ruff for linting and formatting
- Naming: Go standard (camelCase exports, PascalCase types), kebab-case for CLI commands

### Git Workflow

- **Never commit directly to `main`** — always use feature branches
- Branch naming: `<type>/<short-description>` (e.g., `feat/copilot-support`, `fix/session-cleanup`)
- Commits: conventional commits (`feat:`, `fix:`, `docs:`, `ci:`, `refactor:`, `test:`)
- CHANGELOG.md is auto-generated by git-cliff — do NOT edit manually

### File Structure

- `cmd/clincus/` — CLI entry point
- `internal/cli/` — Cobra command definitions
- `internal/config/` — TOML config loading
- `internal/container/` — Incus container management
- `internal/session/` — Session lifecycle
- `internal/server/` — Web dashboard HTTP/WebSocket server
- `internal/health/` — Health check system
- `internal/image/` — Container image builder
- `internal/limits/` — Resource limit enforcement
- `internal/terminal/` — PTY/tmux bridging
- `internal/tool/` — AI tool abstraction (Claude, opencode, Aider)
- `internal/cleanup/` — Orphan resource cleanup
- `web/` — Svelte 5 frontend source
- `webui/` — go:embed wrapper for built frontend assets
- `docs/` — MkDocs Material documentation
- `tests/` — Python integration tests

### Documentation

- Every user-facing change MUST update docs
- CLI changes → `docs/reference/cli.md`
- Config changes → `docs/reference/config.md`
- API changes → `docs/reference/api.md`
- New features → appropriate guide in `docs/guides/`

### Build

- `make build` — builds frontend + Go binary
- `make web` — builds Svelte frontend only
- `webui/dist/.gitkeep` must remain tracked for `go:embed`
- `.gitignore` uses `/clincus` (leading slash) to only ignore root binary

## Anti-Patterns

- **Don't edit CHANGELOG.md** — it's auto-generated by git-cliff from conventional commits
- **Don't rename `incus monitor`** in `internal/server/ws_events.go` — that's the Incus CLI lifecycle monitoring command
- **Don't commit to main** — all changes go through feature branches and PRs
- **Don't use golangci-lint v1 config** — project uses v2 format requiring `golangci-lint-action@v7`
