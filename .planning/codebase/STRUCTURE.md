## Directory Purposes

**`cmd/clincus/`:**
- Purpose: CLI binary entry point only
- Contains: `main.go` which calls `cli.Execute()`
- Key files: `main.go` (15 lines, minimal)
- Binary built to `./clincus` at repo root

**`internal/cli/`:**
- Purpose: Cobra command definitions, flag parsing, orchestration
- Contains: Root command, 14+ subcommands, flag configuration
- Key files:
  - `root.go` — Root command, global flags, PersistentPreRunE hook for config loading
  - `shell.go` — Interactive session command (largest, ~300+ lines)
  - `run.go` — Ephemeral command execution
  - `list.go` — List active sessions
  - `build.go` — Build clincus image
  - `serve.go` — Start web dashboard
  - `attach.go`, `container.go`, `file.go` — Container-specific operations
  - Others: kill, clean, health, info, images, mount_parser, persist, resume, snapshot, shutdown, tmux

**`internal/config/`:**
- Purpose: Load and manage TOML configuration from multiple sources
- Contains: Config structs, loader, file watcher, profile system
- Key files:
  - `config.go` — Config struct definitions (flags, sections)
  - `loader.go` — Load from ~/.config/clincus/, /etc/clincus/, project .clincus.toml
  - `watcher.go` — File system watcher for hot-reload
  - `reload.go` — Config reload logic
  - Test files: `*_test.go`

**`internal/container/`:**
- Purpose: Abstract Incus container operations
- Contains: Manager interface wrapping Incus CLI calls
- Key files:
  - `manager.go` — Manager struct and public methods (Launch, Stop, Delete, Exec, etc.)
  - `commands.go` — Raw Incus command execution helpers (IncusExec, IncusOutput, etc.)
  - Integration tests: `*_integration_test.go` (require running Incus)

**`internal/session/`:**
- Purpose: Session lifecycle, naming, persistence, resume
- Contains: Session ID generation, container naming, setup logic, history
- Key files:
  - `setup.go` — SetupSession() creates container with mounts, limits, tool config
  - `types.go` — Session and mount structures
  - `naming.go` — Container name generation: `clincus-<workspace-hash>-<slot>`
  - `id.go` — Session ID generation and parsing
  - `cleanup.go` — Cleanup containers on exit
  - `paths.go` — Session directory paths
  - `security.go` — Security checks (traversal attacks, protected paths)
  - `mount_validator.go` — Mount path validation
  - `resolve.go` — Resolve resume session ID
  - `history.go` — Session history tracking

**`internal/tool/`:**
- Purpose: Pluggable AI coding tools
- Contains: Tool interface and implementations
- Key files:
  - `tool.go` — Tool interface and Claude implementation
  - `opencode.go` — OpenCode tool implementation
  - `copilot.go` — GitHub Copilot implementation
  - `github.go` — GitHub-specific utilities
  - `registry.go` — Tool registry/factory

**`internal/server/`:**
- Purpose: HTTP and WebSocket server for web dashboard
- Contains: Routes, WebSocket handlers, event hub
- Key files:
  - `server.go` — Server struct, routes, config hot-reload, handlers
  - `api_sessions.go` — Session API handlers (list, create, stop, resume)
  - `api_config.go` — Config API handlers (get, update)
  - `api_workspaces.go` — Workspace API handlers
  - `ws_terminal.go` — WebSocket terminal handler
  - `ws_events.go` — Incus event listener (uses `incus monitor`)
  - `bridge.go` — Terminal I/O bridge (tmux capture ↔ WebSocket)

**`internal/terminal/`:**
- Purpose: PTY and tmux session management
- Contains: Tmux execution, PTY allocation

**`internal/limits/`:**

## Naming Conventions

**Files:**
- Go files: `lowercase_snake_case.go` (e.g., `mount_parser.go`, `ws_terminal.go`)
- Svelte files: `PascalCase.svelte` (e.g., `SessionCard.svelte`)
- TypeScript files: `camelCase.ts` or `kebab-case.ts` for utilities (e.g., `api.ts`, `ws.ts`)
- Test files: `*_test.go` (Go), `conftest.py`, `test_*.py` (Python)
- Integration tests: `*_integration_test.go`

**Directories:**
- Package directories: `lowercase` (no underscores or hyphens, Go convention)
- Feature test directories: `lowercase` (e.g., `tests/shell/`, `tests/limits/`)
- Component directories: `lowercase` (e.g., `web/src/components/`, `web/src/routes/`)

**Types & Constants:**
- Go: PascalCase for exported, camelCase for unexported (Go convention)
- TypeScript: PascalCase for types, interfaces, and enums
- Constants: UPPER_SNAKE_CASE in Go, camelCase in TS

**Functions & Methods:**
- Go: PascalCase (exported) or camelCase (unexported)
- TypeScript: camelCase
- Svelte component methods: camelCase

## Where to Add New Code

**New CLI Command:**
- Create file in `internal/cli/command_name.go`
- Define `var commandNameCmd = &cobra.Command{...}`
- Add to rootCmd in `internal/cli/root.go` line 142-159

**New API Endpoint:**
- Add handler method in `internal/server/api_*.go` (group by domain)
- Register route in `Server.routes()` in `internal/server/server.go` lines 70-83
- Mirror types in `web/src/lib/types.ts`

**New Tool (e.g., support for new AI coding tool):**
- Create `internal/tool/mytool.go` implementing Tool interface
- Register in registry (`internal/tool/registry.go`)
- Update config struct in `internal/config/config.go` if tool-specific config needed

**New Web Component:**
- Create `web/src/components/MyComponent.svelte`
- Import in parent component or route
- Use type definitions from `web/src/lib/types.ts`

**New Session Feature:**
- Add logic to `internal/session/setup.go` for initialization
- Add cleanup logic to `internal/session/cleanup.go`
- Update test in `internal/session/setup_test.go`

**New Resource Limit Type:**
- Add struct to `internal/limits/applier.go` (or new file `internal/limits/types.go`)
- Add validator to `internal/limits/validator.go`
- Add applier function in `internal/limits/applier.go`
- Add CLI flag in `internal/cli/root.go` lines 127-139

**New Configuration Section:**
- Add struct to `internal/config/config.go`
- Update loader in `internal/config/loader.go` to parse section
- Add tests in `internal/config/config_test.go` and `loader_test.go`
- Mirror types in `web/src/lib/types.ts`

## Key Patterns

**Workspace Resolution:**
- Default: Current directory (`.`)
- Resolved to absolute path early in command execution
- Hashed for container naming (workspace.go: `WorkspaceHash()`)
- Mounted to container at `/workspace` or preserved path if configured

**Container Naming:**
- Pattern: `clincus-<workspace-hash>-<slot>`
- Generated by: `session.ContainerName(workspace, slot)`
- Includes workspace and slot for multi-session support in same workspace

**Session Identification:**
- Clincus session ID: UUID or same as container name
- Tool session ID: Tool-specific (Claude: `.claude/sessions-claude/session-uuid`)
- Both tracked for resume functionality

**Mount Organization:**
- Workspace mount: Always mounted to `/workspace` (or preserved path)
- Custom mounts: User-specified via `--mount HOST:CONTAINER`
- Protected mounts: `.git/hooks`, `.git/config`, `.husky`, `.vscode` mounted read-only
- Mount device name: `mount-<index>` (e.g., `mount-0`, `mount-1`)

**Configuration Precedence:**
1. CLI flags (highest priority)
2. Project config (`.clincus.toml`)
3. User config (`~/.config/clincus/config.toml`)
4. System config (`/etc/clincus/config.toml`)
5. Hardcoded defaults (lowest priority)

**Status File Locations:**
- Sessions: `~/.config/clincus/sessions.json` or Incus container metadata
- Tool sessions: `~/.claude/`, `~/.aider/` (tool-specific)
- Logs: `~/.config/clincus/logs/`
- History: `~/.config/clincus/history/` or in-Incus metadata
