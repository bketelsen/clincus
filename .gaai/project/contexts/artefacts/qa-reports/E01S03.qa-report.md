# QA Report: E01S03 — Watch config files and hot-reload on change

## Verdict: PASS

## Per-Criterion Results

| AC | Result | Evidence |
|----|--------|----------|
| AC1 | PASS | `WatchPaths()` returns exactly `/etc/clincus/config.toml` and `~/.config/clincus/config.toml`. `NewWatcher()` uses `fsnotify.NewWatcher()` for filesystem notifications. |
| AC2 | PASS | `ConfigManager.reload()` calls `config.Load()`, acquires write lock, swaps `cm.cfg`. `onChange` callback propagates to server via `srv.UpdateConfig()`. |
| AC3 | PASS | `serve.go` onChange checks `oldPort != newPort` before signaling restart. If port unchanged, no listener action — connections preserved. |
| AC4 | PASS | `listenLoop()` receives new port on `restartCh`, calls `old.Shutdown(ctx)` with 5s timeout, starts new `http.Server` on new port. |
| AC5 | PASS | Incus containers run tmux sessions independently of the Go process. Config reload only affects HTTP server internals. Documented in config.md and web-dashboard.md. |
| AC6 | PASS | `scheduleReload()` uses `time.AfterFunc(w.debounce, ...)` and stops/resets timer on each event. Default debounce 1s. `TestWatcherDebouncing` validates 5 rapid writes produce 1 reload. |
| AC7 | PASS | `reload()`: on `Load()` error, logs message and returns nil. Old config retained. Server continues running. |
| AC8 | PASS | `NewWatcher`: non-existent file → watches parent dir → detects Create event → adds direct file watch. Non-existent parent dir → logs info, skips. Both cases tested. |
| AC9 | PASS | 7 new tests: `TestWatcherDetectsFileModification`, `TestWatcherDebouncing`, `TestWatcherHandlesNonExistentPath`, `TestWatcherHandlesNonExistentParentDir`, `TestConfigManagerGet`, `TestConfigManagerReloadRetainsOnError`, `TestConfigManagerOnChangeCallback`. Existing tests unaffected (zero-value mutex, same Server constructor). |
