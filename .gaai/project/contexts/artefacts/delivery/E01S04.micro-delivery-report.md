# MicroDelivery Report: E01S04 — Notify frontend of config reload events

## Files Changed

| File | Purpose |
|------|---------|
| `internal/server/server.go` | Added `BroadcastConfigReloaded()` method with `config.reloaded` event payload |
| `internal/server/server_test.go` | Added `TestBroadcastConfigReloaded` — verifies event type and timestamp over WebSocket |
| `internal/cli/serve.go` | Wired `BroadcastConfigReloaded()` into the ConfigManager onChange callback |
| `web/src/lib/ws.ts` | Added auto-reconnection to `connectEvents()` with `onReconnect` callback |
| `web/src/App.svelte` | Added `config.reloaded` event handler + `refreshAll` on reconnect |
| `docs/reference/api.md` | Documented `config.reloaded` event type in WebSocket events reference |
| `docs/guides/web-dashboard.md` | Updated hot-reload section with frontend notification behavior |

## Change Summary

Backend: Added `Server.BroadcastConfigReloaded()` that sends `{"type": "config.reloaded", "timestamp": "..."}` to all connected WebSocket clients. Called from the existing ConfigManager onChange callback — only fires on successful reloads.

Frontend: Enhanced `connectEvents()` with automatic WebSocket reconnection (2s delay) and an `onReconnect` callback. App.svelte handles the new event by re-fetching workspace data, and refreshes all data on reconnect.
