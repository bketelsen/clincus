# Story: Notify frontend of config reload events

## User Story

**As a** dashboard user,
**I want** the dashboard to automatically reflect configuration changes when a reload occurs,
**so that** I always see the current active configuration without manually refreshing the page.

---

## Context

The server already has an `EventHub` that broadcasts events to connected WebSocket clients at `/ws/events`. Currently it sends `session.started` and `session.stopped` events from the Incus monitor. This story adds a `config.reloaded` event type that is broadcast when the config watcher (E01S03) completes a successful reload. The frontend listens for this event and re-fetches the config.

---

## Acceptance Criteria

- [ ] AC1: When a config reload succeeds, the server broadcasts a `config.reloaded` event via the existing `/ws/events` WebSocket
- [ ] AC2: The event payload includes a timestamp and optionally indicates which sections changed
- [ ] AC3: The frontend event handler in the Svelte app listens for `config.reloaded` and triggers a re-fetch of `GET /api/config`
- [ ] AC4: If the WebSocket is disconnected during reload, the frontend reconnects and fetches current config on reconnect
- [ ] AC5: Failed reloads do not broadcast an event (only successful reloads notify the frontend)

---

## Out of Scope

- Pushing the full config payload over WebSocket (fetch via API instead)
- Config diff display in the frontend
- Toast/notification UI for reload events (frontend may add this later)

---

## Dependencies

- E01S03 (config watcher must emit reload signal)
- E01S01 (API endpoint must return updated config)

---

## Notes

The existing `EventHub.Broadcast()` method accepts `any` — the new event type can follow the same pattern as session events. The frontend WebSocket handler in the Svelte app already processes event types via a switch/match — adding a new case is straightforward.
