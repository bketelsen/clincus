# Story: Expose full read-only configuration via API

## User Story

**As a** dashboard user,
**I want** an API endpoint that returns the full merged Clincus configuration,
**so that** the frontend can display all active settings without requiring host access.

---

## Context

Currently `GET /api/config` returns only `{port, workspace_roots}`. The full `Config` struct (defaults, limits, security, mounts, incus, tool, git, profiles, dashboard) is loaded at startup but not exposed. A new or expanded endpoint is needed to serve the complete config as read-only JSON.

---

## Acceptance Criteria

- [ ] AC1: `GET /api/config` returns a JSON object containing all top-level config sections: defaults, paths, incus, tool, mounts, limits, git, security, profiles, dashboard
- [ ] AC2: The response uses camelCase or snake_case JSON keys consistently (matching existing API conventions)
- [ ] AC3: The existing `PUT /api/config` endpoint for workspace_roots continues to work unchanged
- [ ] AC4: The endpoint returns the merged config (system + user defaults applied), not raw file contents
- [ ] AC5: Existing Go unit tests pass; new tests cover the expanded response shape

---

## Out of Scope

- Config editing via API (except existing workspace_roots PUT)
- Filtering or partial config queries
- Authentication on the endpoint

---

## Dependencies

- None

---

## Notes

The Go `Config` struct already has JSON-serializable fields. The main work is expanding the API handler in `internal/server/api_config.go` and ensuring the response shape is stable for the frontend to consume.
