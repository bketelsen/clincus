# Story: Add Settings navigation link to dashboard sidebar

## User Story

**As a** dashboard user,
**I want** a visible link to the Settings page in the sidebar navigation,
**so that** I can discover and access the configuration view without manually typing the URL.

---

## Context

Epic E01 delivered a full Settings page at `#/settings` with read-only config display, workspace root management, and hot-reload support. However, the sidebar (`SessionList.svelte`) only contains a home button and session list — there is no link to the Settings page. Users cannot discover the feature without knowing the URL.

---

## Acceptance Criteria

- [ ] AC1: The sidebar displays a Settings navigation link (icon or text) that navigates to `#/settings`
- [ ] AC2: The Settings link is visually distinct from session entries (it is navigation, not a session)
- [ ] AC3: The Settings link indicates active state when the user is on the `#/settings` route
- [ ] AC4: The existing home button and session list remain unchanged
- [ ] AC5: The frontend builds without TypeScript errors (`npx tsc --noEmit` passes)

---

## Technical Notes

- Hash-based routing is already in place (`App.svelte` parses `window.location.hash`)
- The Settings route and component already exist and work
- This is a sidebar-only change — no new routes, no new components, no backend changes
