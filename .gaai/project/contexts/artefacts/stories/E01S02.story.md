# Story: Display configuration in the web dashboard

## User Story

**As a** dashboard user,
**I want** to see the full Clincus configuration displayed in the Settings page,
**so that** I can understand what defaults, limits, security rules, and tool settings are active without reading TOML files.

---

## Context

The Settings page (`web/src/routes/Settings.svelte`) currently only manages workspace roots. With the expanded API from E01S01, the frontend can now render all config sections in a structured, read-only view alongside the existing editable workspace roots.

---

## Acceptance Criteria

- [ ] AC1: The Settings page displays all config sections (defaults, paths, incus, tool, mounts, limits, git, security, profiles, dashboard) in a structured layout
- [ ] AC2: Config values are read-only — no edit controls except for the existing workspace roots management
- [ ] AC3: The display updates automatically when a `config.reloaded` WebSocket event is received (from E01S04)
- [ ] AC4: TypeScript types are defined for the full config response shape in `web/src/lib/types.ts`
- [ ] AC5: The page renders correctly when optional config sections are empty or use defaults

---

## Out of Scope

- Editing config values from the UI (except workspace roots)
- Config diff or change history display
- Config file path display or raw TOML view

---

## Dependencies

- E01S01 (API endpoint must exist)
- E01S04 (WebSocket event for live updates — can degrade gracefully if not yet implemented)

---

## Notes

Design should group config logically by section with clear labels. Consider collapsible sections for large configs (profiles, mounts). The existing Settings page layout and component patterns should be followed.
