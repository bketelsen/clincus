# Epic: Dashboard users can see and understand their Clincus configuration

## Purpose

Operators managing AI coding sessions need visibility into the active configuration — defaults, limits, security, mounts, tool settings — directly from the web dashboard, without SSH-ing into the host or reading TOML files. This reduces misconfiguration risk and accelerates troubleshooting.

The dashboard should also react to configuration changes on disk (system and user config files) by hot-reloading within the same process, so operators don't need to manually restart the server after editing config.

---

## Scope

- Read-only display of the full merged configuration in the web dashboard
- WorkspaceRoots editable from the dashboard (existing capability, retained)
- File watching on system and user config files with in-process hot-reload
- Frontend notification when config reloads
- Validation that active sessions (Incus containers + tmux) are unaffected by reload

---

## Out of Scope

- Editing configuration values other than WorkspaceRoots from the dashboard
- Watching project-level config (`.clincus.toml`) or env-var-specified config
- Persisting WorkspaceRoots edits back to TOML files on disk
- Authentication or authorization for config endpoints

---

## Stories

> Stories are tracked in the backlog. This list is descriptive only.

- E01S01: Expose full read-only configuration via API
- E01S02: Display configuration in the web dashboard
- E01S03: Watch config files and hot-reload on change

---

## Success Metrics

- Dashboard shows all active configuration sections (defaults, limits, security, mounts, tool, incus, git, profiles, dashboard)
- Editing a watched config file triggers automatic reload without manual server restart
- Active terminal sessions remain connected through a config reload cycle
- Frontend reflects updated config within seconds of a file change
