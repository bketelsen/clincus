# Memory Delta: E01S03

## New Patterns

- **Config hot-reload pattern:** `ConfigManager` wraps `config.Load()` with `sync.RWMutex` and fsnotify watcher. Handlers access config via `GetConfig()` read-lock. Mutations via `UpdateConfig()` write-lock. This pattern should be reused for any future runtime-configurable subsystems.
- **Debounced file watcher:** `time.AfterFunc` with reset on each event. 1-second default. Handles editor temp-file-rename patterns (vim, nano).
- **Graceful HTTP listener cycling:** `http.Server.Shutdown()` + goroutine re-listen pattern in `listenLoop()`. Supports port changes without process restart.

## Decisions

- fsnotify v1.8.0 chosen as the file-watching library (only production-viable cross-platform option for Go).
- Project config (`.clincus.toml`) and `CLINCUS_CONFIG` are intentionally not watched (per story out-of-scope).
- Non-existent config paths: watch parent directory and detect file creation, rather than polling.

## No Impact On

- Existing CLI commands — config is still loaded once at startup for CLI use.
- Container lifecycle — completely independent of server process.
